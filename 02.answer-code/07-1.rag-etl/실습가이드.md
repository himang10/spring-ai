# Spring AI RAG ETL 실습 가이드

이 가이드는 Spring AI를 사용하여 다양한 문서(JSON, Text, HTML, PDF, DOCX, URL)를 읽어오고(Read), 이를 벡터 저장소(PGVector)로 변환 및 적재(ETL)하는 과정을 실습하는 절차를 설명합니다.

## 1. 사전 준비 사항

실습을 진행하기 위해 다음 도구들이 설치되어 있어야 합니다.

*   **Java 21** 이상
*   **Maven**
*   **Docker** (PostgreSQL + PGVector 실행용)
*   **OpenAI API Key**

## 2. 데이터베이스 실행 (PGVector)

이 애플리케이션은 벡터 데이터를 저장하기 위해 **PGVector** 확장이 포함된 PostgreSQL을 사용합니다. Docker를 사용하여 데이터베이스를 실행합니다.


> **참고**: 애플리케이션 실행 시 `rag_vector_store` 테이블이 자동으로 생성됩니다 (`application.yml` 설정 참조).

## 3. 애플리케이션 실행

### 3.1 OpenAI API Key 설정

OpenAI API 키를 환경 변수로 설정해야 합니다.

```bash
export OPEN_AI_KEY=sk-your-api-key-here
```

### 3.2 애플리케이션 시작

프로젝트 루트 경로에서 제공된 스크립트 또는 Maven 명령어로 애플리케이션을 실행합니다.

**방법 1: 실행 스크립트 사용**
```bash
./run.sh
```

**방법 2: Maven 직접 실행**
```bash
mvn spring-boot:run
```

애플리케이션이 정상적으로 실행되면 로그에 `Started MyFirstSpringAiApplication` 메시지가 표시됩니다.

## 4. 실습 절차 (웹 인터페이스)

브라우저를 열고 다음 주소로 접속합니다:

*   **URL**: [http://localhost:8080](http://localhost:8080)

화면 상단에 **"🔀 RAG 선택"** 드롭다운 메뉴가 있습니다. 이 메뉴를 통해 다양한 문서 리더와 ETL 파이프라인을 테스트할 수 있습니다.

### 실습 시나리오 1: 문서 읽기 (Reader)

각 파일 형식별로 문서를 단순히 읽어와서 내용을 확인하는 기능입니다.

1.  **RAG 선택** 메뉴에서 **"JSON Reader"** (또는 Text, HTML, PDF 등)를 선택합니다.
2.  하단 입력창에 아무 메시지나 입력하거나(예: "test"), 그냥 **전송 버튼(비행기 아이콘)**을 클릭합니다.
    *   *참고: Reader 모드에서는 입력된 질문이 실제 로직에 영향을 주지 않고 문서 내용 전체를 로드합니다.*
3.  **결과 확인**:
    *   화면에 해당 문서(`src/main/resources/documents/` 경로의 파일)의 내용이 텍스트로 출력되는지 확인합니다.
    *   "문서 로딩 완료: X개의 문서를 찾았습니다." 메시지와 함께 문서 내용이 표시됩니다.

### 실습 시나리오 2: ETL 파이프라인 실행 (ETL to PGVector)

문서를 읽어서 텍스트를 분할(Split)하고, 임베딩(Embedding)하여 벡터 저장소(PGVector)에 저장하는 과정입니다.

1.  **RAG 선택** 메뉴에서 **"JSON Reader (ETL to PGVector)"** (또는 다른 형식의 ETL 옵션)를 선택합니다.
2.  **전송 버튼**을 클릭합니다.
3.  **결과 확인**:
    *   "ETL 파이프라인 완료: ... PGVector에 저장했습니다." 라는 성공 메시지가 표시되는지 확인합니다.
    *   이 과정이 완료되면, 해당 문서의 데이터가 벡터 DB에 저장되어 검색 가능한 상태가 됩니다.

### 지원되는 파일 형식 및 기능

| 형식 | Reader 옵션 (읽기 테스트) | ETL 옵션 (벡터 DB 저장) | 대상 파일 (예시) |
| :--- | :--- | :--- | :--- |
| **JSON** | JSON Reader | JSON Reader (ETL...) | `대한민국헌법.json` |
| **Text** | Text Reader | Text Reader (ETL...) | `대한민국헌법.txt` |
| **HTML** | HTML Reader | HTML Reader (ETL...) | `대한민국헌법.html` |
| **PDF** | PDF Reader | PDF Reader (ETL...) | (resources/documents 내 PDF) |
| **DOCX** | DOCX Reader | DOCX Reader (ETL...) | (resources/documents 내 DOCX) |
| **URL** | URL Reader | URL Reader (ETL...) | (코드에 설정된 URL) |

## 5. 문제 해결

*   **DB 연결 오류**: Docker 컨테이너가 실행 중인지 확인하세요 (`docker ps`).
*   **API Key 오류**: `OPEN_AI_KEY` 환경 변수가 올바르게 설정되었는지 확인하세요.
*   **파일 없음 오류**: `src/main/resources/documents/` 경로에 테스트할 파일들이 존재하는지 확인하세요.
