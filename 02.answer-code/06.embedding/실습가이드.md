# Spring AI Embedding 실습 가이드

## 📚 실습 목표
Spring AI의 Embedding 기능을 이해하고, 텍스트 임베딩과 Vector Store를 활용한 문서 검색 시스템을 구현하면서 벡터 기반 유사도 검색 방법을 학습합니다.

---

## 🎯 실습 전 준비사항

### 1. 프로젝트 설정 확인
```bash
cd 05.embedding
mvn clean install
```

### 2. 필수 개념 이해
- **Embedding**: 텍스트를 고차원 벡터로 변환하는 과정
- **Vector Store**: 임베딩 벡터를 저장하고 유사도 검색을 수행하는 데이터베이스
- **EmbeddingModel**: 텍스트를 벡터로 변환하는 AI 모델
- **Similarity Search**: 벡터 간 유사도를 계산하여 관련 문서를 찾는 검색 방식
- **Document**: 메타데이터와 함께 저장되는 텍스트 단위

### 3. 환경 변수 설정
```bash
export OPEN_AI_KEY=your-openai-api-key
```

### 4. PostgreSQL + PGVector 실행 (Vector Store용)
```bash
./pgvector-run.sh

```

## 📝 실습 1: 텍스트 임베딩 기초

### 목표
EmbeddingModel을 사용하여 텍스트를 벡터로 변환하고, 임베딩의 기본 개념을 이해합니다.

### 학습 포인트
- **EmbeddingModel**: OpenAI의 text-embedding 모델을 사용한 텍스트 벡터화
- **EmbeddingResponse**: 임베딩 결과와 메타데이터를 포함하는 응답 객체
- **Vector Dimensions**: 벡터의 차원 수 (예: 1536차원)
- **임베딩의 활용**: 유사도 검색, 클러스터링, 분류 등

### 실습 단계

#### Step 1: TextEmbeddingService 파일 확인
```bash
cat src/main/java/com/example/springai/service/TextEmbeddingService.java
```

#### Step 2: 기본 구조 이해
```java
package com.example.springai.service;

import java.util.List;

import org.springframework.ai.embedding.Embedding;
import org.springframework.ai.embedding.EmbeddingModel;
import org.springframework.ai.embedding.EmbeddingResponse;
import org.springframework.ai.embedding.EmbeddingResponseMetadata;
import org.springframework.stereotype.Service;

import lombok.extern.slf4j.Slf4j;

@Service
@Slf4j
public class TextEmbeddingService {

  private final EmbeddingModel embeddingModel;

  TextEmbeddingService(EmbeddingModel embeddingModel) {
    this.embeddingModel = embeddingModel;
  }

  public void textEmbedding(String question) {
    // 임베딩하기
    EmbeddingResponse response = embeddingModel.embedForResponse(List.of(question));

    // 임베딩 모델 정보 얻기
    EmbeddingResponseMetadata metadata = response.getMetadata();
    log.info("모델 이름: {}", metadata.getModel());
    log.info("모델의 임베딩 차원: {}", embeddingModel.dimensions());

    // 임베딩 결과 얻기
    Embedding embedding = response.getResults().get(0);
    log.info("벡터 차원: {}", embedding.getOutput().length);
    log.info("벡터: {}", embedding.getOutput());
  }    
}
```

## 📝 실습 2: Vector Store를 활용한 문서 검색

### 목표
PGVector를 사용하여 문서를 벡터 저장소에 저장하고, 유사도 기반 검색을 통해 관련 문서를 찾는 시스템을 구현합니다.

### 학습 포인트
- **VectorStore**: 임베딩 벡터를 저장하는 벡터 데이터베이스
- **PGVector**: PostgreSQL 기반 벡터 저장소
- **Document**: 메타데이터와 함께 저장되는 텍스트 단위
- **Similarity Search**: 벡터 간 코사인 유사도를 계산하여 관련 문서 검색
- **SearchRequest**: topK, similarityThreshold, filterExpression 등을 설정하는 검색 요청

### 실습 단계

#### Step 1: pom.xml 의존성 확인
```xml

    <artifactId>spring-ai-autoconfigure-vector-store-pgvector</artifactId>

    <groupId>org.postgresql</groupId>
```

#### Step 3: DocumentEmbeddingService 구조 이해
```java
@Service
@Slf4j
public class DocumentEmbeddingService {
  private final VectorStore vectorStore;

  DocumentEmbeddingService(VectorStore vectorStore) {
    this.vectorStore = vectorStore;
  }

  // 문서 추가
  public void addDocuments() {
    List<Document> documents = List.of(
        new Document("spring은 web application을 쉽게 만들기 위한 프레임워크입니다.", 
            Map.of("category", "spring", "year", 1987)),
        new Document("spring ai는 AI 모델과의 통합을 단순화하는 프레임워크입니다.", 
            Map.of("category", "spring", "year", 2021)),
        // ... 더 많은 문서들
    );
    vectorStore.add(documents);
  }
  
  // 유사도 검색
  public List<Document> search(String question) {
    List<Document> documents = vectorStore.similaritySearch(
        SearchRequest.builder()
            .query(question)
            .topK(3)
            .build());
    return documents;
  }

  // 필터 적용 검색
  public List<Document> searchWithFilter(String question) {
    List<Document> documents = vectorStore.similaritySearch(
        SearchRequest.builder()
            .query(question)
            .topK(1)
            .similarityThreshold(0.3)
            .filterExpression("category == 'cloud' && year >= 2020")
            .build());
    return documents;
  }

  // 유사 문서 삭제
  public void deleteDocumentForSimilarity(String request) {
    List<Document> similarDocuments = vectorStore.similaritySearch(
        SearchRequest.builder()
            .query(request)
            .topK(1)
            .build());
    
    if (!similarDocuments.isEmpty()) {
      Document docToDelete = similarDocuments.get(0);
      vectorStore.delete(List.of(docToDelete.getId()));
    }
  }

  // 전체 문서 삭제
  public void deleteDocuments() {
    vectorStore.delete("year >= 1987");
  }
}
```

## 🎓 실습 3: 실전 활용 시나리오

### 텍스트 임베딩 vs 문서 검색 비교

| 특징 | 텍스트 임베딩 | 문서 검색 (Vector Store) |
|------|-----------|--------------|
| **목적** | 텍스트의 벡터 표현 생성 | 관련 문서 검색 및 저장 |
| **저장소** | 없음 (즉시 변환) | PostgreSQL + PGVector |
| **결과** | 벡터 배열 (float[]) | Document 객체 리스트 |
| **활용** | 유사도 계산, 클러스터링 | RAG, 지식 검색, 추천 시스템 |
| **성능** | 매우 빠름 | 검색 시 약간 느림 |
| **적합한 용도** | 실시간 유사도 비교 | 대규모 문서 저장 및 검색 |

### 실습 시나리오

#### 시나리오 1: 단순 텍스트 벡터화
```
목적: 두 문장의 유사도를 계산하기 위해 벡터로 변환

1. "Spring AI는 훌륭합니다" → 벡터 A 생성
2. "Spring AI는 좋습니다" → 벡터 B 생성
3. 코사인 유사도 계산 → 0.95 (매우 유사)
```

#### 시나리오 2: 문서 저장 및 검색
```
목적: 여러 프레임워크 문서를 저장하고 관련 문서 찾기
```

### 실행 플로우 이해하기

```
┌─────────────────────────────────────────────────────────────┐
│  텍스트 임베딩 플로우                                          │
└─────────────────────────────────────────────────────────────┘

[1] 사용자가 텍스트 입력
  ↓
[2] EmbeddingModel.embedForResponse() 호출
  ↓
[3] OpenAI API에 임베딩 요청
  - 모델: text-embedding-3-small
  - 차원: 1536
  ↓
[4] 벡터 응답 수신
  - float[] 형태의 1536차원 벡터
  ↓
[5] 콘솔에 벡터 정보 출력
  - 모델명, 차원, 벡터값


┌─────────────────────────────────────────────────────────────┐
│  문서 검색 플로우 (Vector Store)                              │
└─────────────────────────────────────────────────────────────┘

[1] 문서 추가 요청
  ↓
[2] Document 리스트 생성
  - 텍스트 + 메타데이터 (category, year)
  ↓
[3] VectorStore.add(documents) 호출
  - 각 문서를 임베딩 벡터로 변환
  - PostgreSQL에 저장 (id, content, metadata, embedding)
  ↓
[4] 저장 완료

[5] 검색 요청
  ↓
[6] 검색 쿼리를 벡터로 변환
  ↓
[7] VectorStore.similaritySearch() 호출
  - 쿼리 벡터와 저장된 벡터들 간 코사인 유사도 계산
  - topK개의 가장 유사한 문서 선택
  - 필터 조건 적용 (옵션)
  ↓
[8] Document 리스트 반환
  - content, metadata 포함
```

## 🔧 문제 해결 가이드

### 문제 1: VectorStore 빈을 찾을 수 없음
```
Error creating bean with name 'documentEmbeddingService': 
Unsatisfied dependency expressed through constructor parameter 0: 
No qualifying bean of type 'org.springframework.ai.vectorstore.VectorStore'
```

**해결 방법:**
1. `spring-ai-autoconfigure-vector-store-pgvector` 의존성 확인
2. PostgreSQL 실행 확인: `docker ps | grep pgvector`
3. `application.yml`의 datasource 설정 확인

### 문제 2: PGVector 확장 오류
```
ERROR: extension "vector" does not exist
```

**해결 방법:**
```sql
-- PostgreSQL 접속
docker exec -it pgvector psql -U postgres -d postgres

-- vector 확장 설치
CREATE EXTENSION IF NOT EXISTS vector;

-- 확인
\dx
```

### 문제 3: 테이블이 생성되지 않음
```
ERROR: relation "vector_store" does not exist
```

**해결 방법:**
- `application.yml`의 `schema-validation: true` 확인
- 애플리케이션 재시작
- 수동으로 테이블 생성 (필요시):
```sql
CREATE TABLE vector_store (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  content TEXT,
  metadata JSONB,
  embedding VECTOR(1536)
);
```

### 문제 4: EmbeddingModel 빈을 찾을 수 없음
```
Error creating bean with name 'textEmbeddingService': 
No qualifying bean of type 'org.springframework.ai.embedding.EmbeddingModel'
```

**해결 방법:**
1. `spring-ai-starter-model-openai` 의존성 확인
2. `OPEN_AI_KEY` 환경 변수 설정 확인
3. `application.yml`의 embeddings 설정 확인

### 문제 5: 검색 결과가 없음
```
Search returned no results
```

**해결 방법:**
1. 먼저 "Add Documents" 버튼으로 문서 추가
2. PostgreSQL에서 데이터 확인:
```sql
SELECT COUNT(*) FROM vector_store;
SELECT content FROM vector_store LIMIT 3;
```
3. 검색 쿼리를 더 일반적으로 변경
4. `similarityThreshold` 값을 낮춤 (예: 0.3 → 0.1)

## 🚀 추가 학습 자료

### 1. Embedding 모델 종류
- **text-embedding-3-small**: 1536차원, 빠르고 경제적
- **text-embedding-3-large**: 3072차원, 더 정확한 임베딩
- **text-embedding-ada-002**: 레거시 모델 (1536차원)

### 2. VectorStore 옵션
- **PGVector**: PostgreSQL 확장, 프로덕션 환경에 적합
- **Qdrant**: 전문 벡터 데이터베이스, 고성능
- **Pinecone**: 클라우드 벡터 데이터베이스, 관리 불필요
- **Chroma**: 경량 벡터 데이터베이스, 로컬 개발에 적합

### 3. 거리 계산 방식
```yaml
# application.yml
spring.ai.vectorstore.pgvector:
  distance-type: COSINE_DISTANCE  # 기본값, 방향 유사도
  # distance-type: EUCLIDEAN_DISTANCE  # 유클리드 거리
  # distance-type: NEGATIVE_INNER_PRODUCT  # 내적
```
