# Spring AI Advisor 개발 실습 가이드

## 📚 실습 목표
Spring AI의 Advisor 패턴을 이해하고, 단계별로 3가지 Advisor를 직접 구현하면서 AOP 방식의 LLM 요청/응답 제어 방법을 학습합니다.

---

## 🎯 실습 전 준비사항

### 1. 프로젝트 설정 확인
```bash
cd 04.advisors
mvn clean install
```

### 2. 필수 개념 이해
- **Advisor 패턴**: AOP(Aspect Oriented Programming)와 유사하게 LLM 호출 전후에 로직을 삽입
- **Advisor Chain**: 여러 Advisor가 순차적으로 실행되는 체인 구조
- **Order**: 작은 값일수록 먼저 실행 (0 -> 1 -> 2)

## 🎓 실습 4: Advisor 통합 및 순서 제어 (난이도: ⭐⭐)

### 목표
여러 Advisor를 조합하여 실행 순서를 제어하고, 각 Advisor가 협력하는 방식을 이해합니다.

### 실습 단계

#### Step 1: ChatClientConfig에서 Advisor 등록
```java
@Configuration
public class ChatClientConfig {
    
    @Bean
    public SimpleLoggingAdvisor simpleLoggingAdvisor() {
        return new SimpleLoggingAdvisor();
    }
    
    @Bean
    public JsonLoggingAdvisor jsonLoggingAdvisor(ObjectMapper objectMapper) {
        return new JsonLoggingAdvisor(objectMapper, 0);
    }
    
    @Bean
    public MaxCharLengthAdvisor maxCharLengthAdvisor() {
        return new MaxCharLengthAdvisor(10);  // order = 10
    }
    
    @Bean
    public ChatClient chatClient(ChatClient.Builder builder, 
                                   JsonLoggingAdvisor jsonLoggingAdvisor) {
        return builder
            .defaultAdvisors(jsonLoggingAdvisor)  // 기본 Advisor
            .build();
    }
}
```

#### Step 2: Service에서 여러 Advisor 조합
```java
@Service
public class AdvisorService {
    
    private final ChatClient chatClient;
    private final SimpleLoggingAdvisor simpleLoggingAdvisor;
    private final MaxCharLengthAdvisor maxCharLengthAdvisor;
    
    // TODO: 생성자 주입
    
    public String callWithMultipleAdvisors(String message) {
        Map<String, Object> context = new HashMap<>();
        context.put(MaxCharLengthAdvisor.MAX_CHAR_LENGTH, 100);
        
        return chatClient.prompt()
            .user(message)
            .advisors(
                simpleLoggingAdvisor,     // order = 0
                maxCharLengthAdvisor      // order = 10
            )
            .advisorContext(context)
            .call()
            .content();
    }
}
```

### 실행 순서 이해하기

```
┌─────────────────────────────────────────────────────────────┐
│  Advisor 체인 실행 순서 (order 값 기준)                      │
└─────────────────────────────────────────────────────────────┘

요청 단계 (Request Phase):
  ↓
[1] JsonLoggingAdvisor (order=0, defaultAdvisor)
  → 요청 로깅
  ↓
[2] SimpleLoggingAdvisor (order=0)
  → 간단한 요청 로깅
  ↓
[3] MaxCharLengthAdvisor (order=10)
  → 메시지에 "100자 이내로..." 추가
  ↓
[4] LLM API 호출
  ↓
응답 단계 (Response Phase):
  ↓
[3] MaxCharLengthAdvisor
  → 응답 반환
  ↓
[2] SimpleLoggingAdvisor
  → 간단한 응답 로깅
  ↓
[1] JsonLoggingAdvisor
  → 응답 로깅
  ↓
최종 응답 반환
```

- Context에서 `targetLanguage` 값을 받아 동적으로 언어 지정

### 도전 2: Error Handling Advisor
LLM 호출 시 에러가 발생하면 재시도하는 `RetryAdvisor`를 구현하세요:
- 최대 3번 재시도
- 각 재시도마다 1초 대기
- 실패 시 기본 응답 반환

### 도전 3: Performance Monitoring Advisor
LLM 호출 시간을 측정하고 통계를 수집하는 `PerformanceAdvisor`를 구현하세요:
- 요청 시작/종료 시간 기록
- 평균 응답 시간 계산
- 느린 요청(3초 이상) 경고 로그
