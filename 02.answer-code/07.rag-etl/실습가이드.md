# Spring AI Chat Memory 실습 가이드

## 📚 실습 목표
Spring AI의 Chat Memory 기능을 이해하고, In-Memory와 Vector Store 기반의 두 가지 대화 메모리 방식을 구현하면서 LLM과의 상태 유지 대화 방법을 학습합니다.

---

## 🎯 실습 전 준비사항

### 1. 프로젝트 설정 확인
```bash
cd 05.chat-memory
mvn clean install
```

### 2. 필수 개념 이해
- **Chat Memory**: LLM이 대화 컨텍스트를 유지하고 활용하는 정보
- **Chat History**: 사용자와 모델 간 교환된 전체 대화 기록
- **Conversation ID**: 대화 세션을 구분하는 고유 식별자
- **ChatMemoryRepository**: 메시지 저장 및 검색을 담당하는 저장소
- **ChatMemoryAdvisor**: ChatClient에서 자동으로 메모리를 관리하는 컴포넌트

### 3. 환경 변수 설정
```bash
export OPEN_AI_KEY=your-openai-api-key
```

### 4. PostgreSQL + PGVector 실행 (Vector Store용)
```bash
docker run -d \
  --name pgvector \
  -e POSTGRES_PASSWORD=postgres \
  -p 5432:5432 \
  pgvector/pgvector:pg16
```

## 📝 실습 2: Vector Store Chat Memory 구현

### 목표
PGVector를 사용하여 대화 이력을 벡터 저장소에 저장하고, 의미 기반 검색으로 관련 대화를 가져오는 고급 Chat Memory를 구현합니다.

### 학습 포인트
- **VectorStore**: 임베딩 벡터를 저장하는 벡터 데이터베이스
- **PGVector**: PostgreSQL 기반 벡터 저장소
- **VectorStoreChatMemoryAdvisor**: VectorStore 기반 메모리 관리
- **Semantic Search**: 의미 기반 대화 검색

### 실습 단계

#### Step 1: pom.xml 의존성 확인
```xml

</dependency>

    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-autoconfigure-vector-store-pgvector</artifactId>
```

## 🎓 실습 3: 두 가지 방식 비교 및 활용

### In-Memory vs Vector Store 비교

| 특징 | In-Memory | Vector Store |
|------|-----------|--------------|
| **저장소** | 메모리 (ConcurrentHashMap) | PostgreSQL + PGVector |
| **지속성** | 재시작 시 소실 | 영구 저장 |
| **검색 방식** | 순차 검색 (최근 N개) | 의미 기반 벡터 검색 |
| **성능** | 빠름 | 검색 시 약간 느림 |
| **확장성** | 메모리 제약 | 디스크 제약 (높음) |
| **적합한 용도** | 단기 대화, 프로토타입 | 장기 대화, 프로덕션 |

### 실습 시나리오

#### 시나리오 1: 짧은 대화 테스트 (In-Memory)
```
1. "너의 이름은 SKALA AI 입니다." 
2. "너의 이름은?" 
   → 최근 메시지를 모두 포함하여 답변

3. "내가 조금전에 무엇을 말했나?"
   → 메모리 윈도우 내의 모든 메시지 참조 가능
```

#### 시나리오 2: 긴 대화 테스트 (Vector Store)
```
1. "나의 이름은 김철수입니다."
2. "나는 서울에 살아요."
3. "나의 취미는 등산입니다."
... (여러 대화 진행)
20. "나의 이름이 뭐였지?"
    → 벡터 검색으로 1번 메시지를 찾아 답변
```

#### 시나리오 3: 의미 기반 검색 (Vector Store)
```
1. "나는 강아지를 키워요."
2. "나는 고양이도 좋아해요."
3. "주말에는 산책을 가요."
...
10. "나는 어떤 동물을 키우나요?"
    → "강아지"와 "고양이" 메시지를 벡터 검색으로 찾아 답변
```

### 실행 플로우 이해하기

```
┌─────────────────────────────────────────────────────────────┐
│  In-Memory Chat Memory 플로우                                │
└─────────────────────────────────────────────────────────────┘

[1] User 메시지 입력
  ↓
[2] MessageChatMemoryAdvisor 실행
  - ChatMemory.get(conversationId) 호출
  - InMemoryChatMemoryRepository에서 메시지 조회
  - 최근 N개 메시지 가져오기
  ↓
[3] 가져온 메시지들을 Prompt에 추가
  ↓
[4] LLM API 호출
  ↓
[5] LLM 응답 수신
  ↓
[6] MessageChatMemoryAdvisor가 응답 저장
  - ChatMemory.add(conversationId, assistantMessage)
  - InMemoryChatMemoryRepository에 저장
  ↓
[7] 최종 응답 반환


┌─────────────────────────────────────────────────────────────┐
│  Vector Store Chat Memory 플로우                             │
└─────────────────────────────────────────────────────────────┘

[1] User 메시지 입력
  ↓
[2] VectorStoreChatMemoryAdvisor 실행
  - VectorStore.similaritySearch(userMessage, conversationId)
  - 메시지를 임베딩 벡터로 변환
  - 유사한 벡터 검색 (코사인 유사도)
  - 관련 대화 이력 조회
  ↓
[3] 검색된 관련 대화를 System Prompt에 텍스트로 추가
  ↓
[4] LLM API 호출
  ↓
[5] LLM 응답 수신
  ↓
[6] VectorStoreChatMemoryAdvisor가 응답 저장
  - User/Assistant 메시지를 벡터로 변환
  - VectorStore.add(documents) - PostgreSQL에 저장
  ↓
[7] 최종 응답 반환
```

## 🔧 문제 해결 가이드

### 문제 1: VectorStore 빈을 찾을 수 없음
```
Error creating bean with name 'vectorStoreService': 
Unsatisfied dependency expressed through constructor parameter 0: 
No qualifying bean of type 'org.springframework.ai.vectorstore.VectorStore'
```

**해결 방법:**
1. `spring-ai-autoconfigure-vector-store-pgvector` 의존성 확인
2. PostgreSQL 실행 확인: `docker ps | grep pgvector`
3. `application.yml`의 datasource 설정 확인

### 문제 2: PGVector 확장 오류
```
ERROR: extension "vector" does not exist
```

**해결 방법:**
```sql
-- PostgreSQL 접속
docker exec -it pgvector psql -U postgres -d postgres

-- vector 확장 설치
CREATE EXTENSION IF NOT EXISTS vector;

-- 확인
\dx
```

### 문제 3: 테이블이 생성되지 않음
```
ERROR: relation "chat_memory_vector_store" does not exist
```

**해결 방법:**
- `application.yml`의 `schema-validation: true` 확인
- 애플리케이션 재시작
- 수동으로 테이블 생성 (필요시)
