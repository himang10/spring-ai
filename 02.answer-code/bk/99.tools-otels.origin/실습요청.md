# Spring AI Tools 실습 가이드

이 실습은 Spring AI에서 Function Calling (Tools)을 활용하여 AI가 외부 기능을 호출할 수 있도록 구현하는 과정을 학습합니다.

## 🎯 학습 목표

1. Spring AI의 Controller-Service-Tools 아키텍처 이해
2. AI가 호출할 수 있는 Tool 메서드 설계 및 구현
3. `@Tool`, `@ToolParam` 어노테이션을 활용한 Function Calling 구현
4. Claude AI 모델과 연동하여 실시간 정보 제공

## 📝 1단계: 기본 구조 구현 (Tools 없이)

> **목표**: Controller → Service → Tools 기본 구조를 만들되, AI Tool 기능은 제외하고 일반 메서드로 구현

### 1.1 DateTime 패키지 구현

#### 🔹 DateTimeController 작성

**구현 요구사항:**
- REST 컨트롤러로 설정 (`@RestController`)
- 엔드포인트: `/chat/date-time-tools` (POST)
- 파라미터: `question` (String) - 사용자 질문 입력
- 반환 타입: `String` (일반 텍스트)
- `DateTimeService`를 주입받아 사용

**힌트:**
```java
@PostMapping(
    value = "/date-time-tools",
    consumes = MediaType.APPLICATION_FORM_URLENCODED_VALUE,
    produces = MediaType.TEXT_PLAIN_VALUE
)
```

#### 🔹 DateTimeService 작성

**구현 요구사항:**
- 서비스 계층 (`@Service`)
- `ChatClient.Builder`를 생성자 주입으로 받아 `ChatClient` 빌드
- `DateTimeTools` 의존성 주입 (2단계를 위해 미리 준비)
- `chat(String question)` 메서드 구현
  - 현재는 ChatClient를 사용하지 않고 간단한 응답 반환
  - 예: `"현재 시간 기능을 구현 중입니다."`

**핵심 포인트:**
- ChatClient는 Spring AI에서 LLM과 통신하는 클라이언트
- Builder 패턴으로 구성하여 나중에 확장 가능하게 설계

#### 🔹 DateTimeTools 작성 (일반 메서드)

**구현 요구사항:**
- 컴포넌트로 등록 (`@Component`)
- 두 개의 메서드 구현:

**1) `getCurrentDateTime()` 메서드**
- 현재 시간을 ISO-8601 형식으로 반환
- 사용자의 시간대(`LocaleContextHolder.getTimeZone()`)를 고려
- `LocalDateTime.now()` 활용

**2) `setAlarm(String time)` 메서드**
- 파라미터: ISO-8601 형식의 시간 문자열
- 현재 시간과 설정한 알람 시간을 함께 반환
- 콘솔에 알람 설정 메시지 출력

**힌트:**
```java
LocalDateTime.now()
    .atZone(LocaleContextHolder.getTimeZone().toZoneId())
    .toString();
```

### 1.2 Weather 패키지 구현

#### 🔹 WeatherController 작성

**구현 요구사항:**
- REST 컨트롤러로 설정
- 엔드포인트: `/chat/weather-tools` (POST)
- 파라미터: `question` (String)
- `WeatherService`를 주입받아 사용
- 로그를 추가하여 요청 추적

#### 🔹 WeatherService 작성

**구현 요구사항:**
- 서비스 계층 (`@Service`)
- `ChatClient.Builder`를 생성자 주입으로 받아 `ChatClient` 빌드
- `WeatherTools` 의존성 주입 (2단계를 위해 미리 준비)
- `chat(String question)` 메서드 구현
  - 현재는 간단한 응답 반환
  - 예: `"날씨 조회 기능을 구현 중입니다."`

#### 🔹 WeatherTools 작성 (일반 메서드)

**구현 요구사항:**
- 컴포넌트로 등록 (`@Component`)
- `WebClient`를 사용하여 OpenWeatherMap API 호출
- 생성자에서 필요한 설정값 주입:
  - `@Value("${weather.api.key}")` - API 키
  - `@Value("${weather.api.base-url}")` - 베이스 URL
  - `@Value("${weather.api.path}")` - API 경로
  - `WebClient.Builder` - HTTP 클라이언트

**`getCurrentWeather(String city)` 메서드 구현:**
- 파라미터: 도시 이름 (예: Seoul, Busan)
- OpenWeatherMap API 호출하여 날씨 정보 조회
- 반환값: JSON 문자열
- URI 파라미터:
  - `q`: 도시 이름
  - `appid`: API 키
  - `units`: "metric" (섭씨 온도)
  - `lang`: "kr" (한국어)

**WebClient 사용 힌트:**
```java
webClient.get()
    .uri(uriBuilder -> uriBuilder
        .path(weatherApiPath)
        .queryParam("q", city)
        .queryParam("appid", weatherApiKey)
        .queryParam("units", "metric")
        .queryParam("lang", "kr")
        .build())
    .retrieve()
    .bodyToMono(String.class)
    .block();
```

**application.yml 설정 예시:**
```yaml
weather:
  api:
    base-url: https://api.openweathermap.org
    path: /data/2.5/weather
    key: YOUR_API_KEY_HERE
```

## 🚀 2단계: Spring AI Tools 적용 (Claude 연동)

> **목표**: `@Tool` 어노테이션을 적용하여 AI가 자동으로 적절한 도구를 선택하고 호출하도록 구현

### 2.1 DateTimeTools에 @Tool 추가

#### 🔹 `getCurrentDateTime()` 메서드 수정

**구현 요구사항:**
- `@Tool` 어노테이션 추가
- `description` 속성: AI가 이 도구를 언제 사용할지 명확히 설명
  - 예: "사용자의 시간대에 맞는 현재 날짜와 시간 정보를 제공합니다."

**핵심 개념:**
- `@Tool` 어노테이션은 Spring AI가 이 메서드를 Function으로 인식하게 함
- description은 AI에게 제공되는 도구 사용 설명서

#### 🔹 `setAlarm(String time)` 메서드 수정

**구현 요구사항:**
- `@Tool` 어노테이션 추가
- description: "주어진 시간에 사용자 알람을 설정합니다. 시간은 ISO-8601 형식입니다..."
- 파라미터에 `@ToolParam` 추가:
  ```java
  @ToolParam(description = "ISO-8601 형식의 시간", required = true)
  String time
  ```

**중요 포인트:**
- `@ToolParam`은 AI에게 파라미터의 의미와 필수 여부를 알려줌
- AI는 이 정보를 바탕으로 적절한 값을 추출하여 전달

### 2.2 WeatherTools에 @Tool 추가

#### 🔹 `getCurrentWeather(String city)` 메서드 수정

**구현 요구사항:**
- `@Tool` 어노테이션 추가
- description: "도시 이름으로 현재 날씨를 조회합니다. 예: Seoul"
- 파라미터에 `@ToolParam` 추가:
  ```java
  @ToolParam(description = "도시 이름 (예: Seoul)", required = true)
  String city
  ```

**주의사항:**
- 도시 이름은 영어로 입력받아야 OpenWeatherMap API가 정확히 인식
- AI가 한글 도시명을 영어로 변환하여 전달

### 2.3 DateTimeService 수정

#### 🔹 chat() 메서드를 ChatClient로 변경

**구현 요구사항:**
- ChatClient를 사용하여 실제 AI 호출
- `.tools(dateTimeTools)` 추가 - AI가 사용할 도구 등록
- `SimpleLoggerAdvisor` 추가 - 로깅 (선택사항)

**구현 예시:**
```java
public String chat(String question) {
    String answer = this.chatClient.prompt()
        .user(question)
        .tools(dateTimeTools)  // 핵심! AI가 사용할 도구 등록
        .call()
        .content();
    return answer;
}
```

**동작 원리:**
1. 사용자 질문을 AI에게 전달
2. AI는 등록된 Tools의 description을 분석
3. 필요하면 적절한 Tool 메서드 호출
4. Tool 실행 결과를 바탕으로 자연어 응답 생성

### 2.4 WeatherService 수정

#### 🔹 chat() 메서드를 ChatClient로 변경

**구현 요구사항:**
- ChatClient를 사용하여 실제 AI 호출
- `.tools(weatherTools)` 추가
- `.system()` 프롬프트 추가 - AI에게 응답 형식 가이드

**시스템 프롬프트 작성 팁:**
```java
.system("""
    날씨 정보를 조회할 때는 다음 형식으로 결과를 보기 좋게 정리해서 사용자에게 제공하세요:
    
    [도시명 날씨 정보]
    - 온도: X°C
    - 체감 온도: X°C
    - 날씨: 날씨 상태
    - 습도: X%
    - 풍속: X m/s
    - 구름: X%
    """)
```

**시스템 프롬프트의 역할:**
- AI의 응답 스타일과 형식을 제어
- 일관된 사용자 경험 제공

## 🎓 핵심 학습 포인트

### 1. @Tool 어노테이션의 역할
- AI가 호출 가능한 함수로 등록
- description이 AI의 도구 선택 기준
- 명확하고 구체적인 설명이 중요

### 2. @ToolParam의 중요성
- AI가 파라미터 의미를 이해하도록 도움
- `required` 속성으로 필수 여부 지정
- AI가 사용자 질문에서 적절한 값 추출

### 3. ChatClient의 .tools() 메서드
- 여러 도구를 등록 가능
- AI가 상황에 맞는 도구를 자동 선택
- 도구 조합 사용도 가능

### 4. Function Calling 플로우
```
사용자 질문 
  → AI가 질문 분석 
  → 적절한 Tool 선택 
  → Tool 메서드 실행 
  → 결과를 바탕으로 자연어 응답 생성
```
