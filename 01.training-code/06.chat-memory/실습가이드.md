# Spring AI Chat Memory 실습 가이드

## 📚 실습 목표
Spring AI의 다양한 Chat Memory 구현 방식을 이해하고, Manual부터 Vector Store 기반까지 단계별로 대화 메모리를 구현하면서 LLM과의 상태 유지 대화 방법을 학습합니다.

---

## 🎯 사전 조건

### 1. PostgreSQL + PGVector 컨테이너 실행
Vector Store 기반 Chat Memory를 위해 pgVector가 실행되어 있어야 합니다.

```bash
docker run -d \
  --name pgvector \
  -e POSTGRES_PASSWORD=postgres \
  -p 5432:5432 \
  pgvector/pgvector:pg16
```

### 2. application.yml 설정 확인
다음 설정이 사전에 정의되어 있어야 합니다:

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/postgres
    username: postgres
    password: postgres
  
  ai:
    openai:
      api-key: ${OPEN_AI_KEY}
    vectorstore:
      pgvector:
        initialize-schema: true
        
server:
  port: 8080
```

### 3. 환경 변수 설정
```bash
export OPEN_AI_KEY=your-openai-api-key
```

### 4. 프로젝트 빌드
```bash
cd 06.chat-memory
mvn clean install
```

## 📝 실습 1: Manual Chat Memory

### 목표
ChatMemory Advisor를 사용하지 않고 수동으로 대화 이력을 관리하는 방식을 이해합니다.

### 학습 포인트
- Advisor 없이 수동으로 메시지 관리
- SystemMessage, UserMessage, AssistantMessage 직접 추가
- ChatMemory.add()와 ChatMemory.get()으로 수동 관리
- MessageWindowChatMemory를 직접 생성하여 사용

### 구현된 코드
- service/ManualChatMemoryService.java
- controller/InMemoryController.java의 `/chat/manual-chat-memory` 엔드포인트

**엔드포인트:** `/chat/manual-chat-memory` 

---

## 📝 실습 2: Default Chat Memory

### 목표
Spring이 자동으로 구성하는 기본 ChatMemory Bean을 사용하여 대화 이력을 자동으로 관리합니다.

### 학습 포인트
- Spring Auto Configuration으로 생성되는 기본 ChatMemory Bean
- InMemoryChatMemory: ConcurrentHashMap 기반 메모리 저장
- MessageChatMemoryAdvisor: 자동으로 메시지 저장/조회
- 별도의 ChatMemory 생성 없이 주입받아 사용

### 구현된 코드
- service/DefaultChatMemoryService.java
- controller/InMemoryController.java의 `/chat/in-memory` 엔드포인트

**엔드포인트:** `/chat/in-memory`

---

## 📝 실습 3: Message Window Chat Memory

### 목표
최근 N개의 메시지만 유지하는 윈도우 방식의 Chat Memory를 명시적으로 구성하여 사용합니다.

### 학습 포인트
- MessageWindowChatMemory 명시적 생성
- maxMessages 설정으로 메시지 개수 제한
- 오래된 메시지 자동 삭제 (윈도우 방식)
- MessageChatMemoryAdvisor와 함께 사용

### 구현된 코드
- service/MessageChatMemoryService.java
- controller/InMemoryController.java의 `/chat/message-chat-memory` 엔드포인트

**엔드포인트:** `/chat/message-chat-memory`

---

## 📝 실습 4: Prompt Chat Memory

### 목표
시스템 프롬프트에 텍스트 형태로 대화 기록을 추가하는 방식을 구현합니다.

### 학습 포인트
- PromptChatMemoryAdvisor: 프롬프트에 텍스트로 추가
- MessageChatMemoryAdvisor와의 차이점 이해
- 텍스트 기반 메모리 전달 방식
- Spring 기본 ChatMemory Bean 주입받아 사용

### 구현된 코드
- service/PromptChatMemoryService.java
- controller/InMemoryController.java의 `/chat/prompt-chat-memory` 엔드포인트

**엔드포인트:** `/chat/prompt-chat-memory`

---

## 📝 실습 5: Vector Store Chat Memory

### 목표
PGVector를 사용하여 대화 이력을 벡터 DB에 저장하고 의미 기반 검색으로 관련 대화를 가져옵니다.

### 학습 포인트
- VectorStore: 임베딩 벡터 저장소
- VectorStoreChatMemoryAdvisor: 벡터 기반 메모리 관리
- Semantic Search: 의미 기반 대화 검색
- 영구 저장 및 확장 가능한 메모리

### 구현된 코드
- service/VectorStoreService.java
- controller/VectorStoreController.java

**엔드포인트:** `/chat/vector-store`

## 🚀 애플리케이션 실행 및 테스트

### 1. 애플리케이션 실행
```bash
mvn spring-boot:run
```

### 2. 브라우저 접속
```
http://localhost:8080
```

### 3. Chat Memory 방식 선택
화면 상단의 드롭다운에서 테스트할 Chat Memory 방식을 선택합니다:
- Manual Chat Memory
- In-Memory Chat Memory(Advisor)
- Message Window Chat Memory(Advisor)
- Prompt Chat Memory(Advisor)
- Vector Store Chat Memory(Advisor)

### 4. 대화 테스트

#### 테스트 시나리오 1: 기본 메모리 테스트
```
1. "너의 이름은 SKALA AI 입니다." 입력
2. "너의 이름은 무엇인가?" 입력
   → "SKALA AI"라고 답변하는지 확인
```

#### 테스트 시나리오 2: 연속 대화 테스트
```
1. "나는 등산을 좋아해"
2. "나는 서울에 살아"
3. "내 취미가 뭐였지?"
   → 1번 메시지 기억하여 "등산"이라고 답변
```

#### 테스트 시나리오 3: 긴 대화 테스트 (Vector Store)
```
1. "나의 이름은 김철수"
2-9. (다양한 대화 진행)
10. "내 이름이 뭐였지?"
    → Vector Store가 유사도 검색으로 1번 메시지를 찾아 답변
```

### 5. 새 대화 시작
우측 상단의 "+" 버튼을 클릭하여 새로운 대화 세션을 시작합니다.

## 🔍 각 방식별 특징 비교

| 방식 | 저장소 | Advisor | 메모리 타입 | 검색 방식 | 사용 사례 |
|------|--------|---------|------------|-----------|-----------|
| **Manual** | MessageWindowChatMemory | ✗ | 최근 N개 | 수동 관리 | 학습/커스텀 로직 |
| **Default** | InMemoryChatMemory | ✓ | 전체 저장 | 전체 조회 | 프로토타입 |
| **MessageWindow** | MessageWindowChatMemory | ✓ | 최근 N개 | 자동 윈도우 | 단기 대화 |
| **Prompt** | InMemoryChatMemory | ✓ (텍스트) | 전체 저장 | 텍스트 변환 | 특수 포맷 |
| **VectorStore** | PostgreSQL+PGVector | ✓ | 영구 저장 | 의미 기반 | 프로덕션 |

## 🔧 문제 해결

### 1. VectorStore Bean 오류
```
No qualifying bean of type 'VectorStore'
```
**해결:** pgvector 컨테이너 실행 확인, application.yml의 datasource 설정 확인

### 2. 포트 충돌
```
Port 8080 is already in use
```
**해결:** `server.port`를 application.yml에서 변경하거나 기존 프로세스 종료

### 3. OpenAI API 키 오류
```
401 Unauthorized
```
**해결:** 환경 변수 `OPEN_AI_KEY` 설정 확인
