# Spring AI Advisor 개발 실습 가이드

## 📚 실습 목표
Spring AI의 Advisor 패턴을 이해하고, 단계별로 2가지 Advisor를 직접 구현하면서 AOP 방식의 LLM 요청/응답 제어 방법을 학습합니다.

---

## 🎯 실습 전 준비사항

### 1. 프로젝트 설정 확인
```bash
cd 05.advisors
mvn clean install
```

### 2. 필수 개념 이해
- **Advisor 패턴**: AOP(Aspect Oriented Programming)와 유사하게 LLM 호출 전후에 로직을 삽입
- **Advisor Chain**: 여러 Advisor가 순차적으로 실행되는 체인 구조
- **Order**: 작은 값일수록 먼저 실행 (Ordered.HIGHEST_PRECEDENCE -> 0 -> 10)
- **Context**: Advisor 간 데이터를 전달하는 Map 구조

## 📝 실습 2: MaxCharLengthAdvisor 구현

### 목표
사용자 메시지를 수정하여 응답 길이를 제한하는 Advisor를 구현합니다.

### 학습 포인트
- **요청 변경(Request Mutation)**: LLM에 전달되는 요청을 수정
- **Context 활용**: 요청별 설정값 전달

### 실습 단계

#### Step 1: 파일 생성 및 필드 정의
```java
package com.example.springai.advisor;

import org.springframework.ai.chat.client.ChatClientRequest;
import org.springframework.ai.chat.client.ChatClientResponse;
import org.springframework.ai.chat.client.advisor.api.CallAdvisor;
import org.springframework.ai.chat.client.advisor.api.CallAdvisorChain;
import org.springframework.ai.chat.messages.UserMessage;
import org.springframework.ai.chat.prompt.Prompt;
import lombok.extern.slf4j.Slf4j;

@Slf4j
public class MaxCharLengthAdvisor implements CallAdvisor {
    
    // TODO: Context에서 사용할 키 상수 정의 (주의: LENGH 철자)
    public static final String MAX_CHAR_LENGH = "maxCharLength";
    
    // TODO: 기본 최대 문자 수 필드 (300자)
    private int maxCharLength = 300;
    
    // TODO: 실행 순서 필드
    private int order;
    
    // TODO: 생성자 구현 (order를 받아서 초기화)
    public MaxCharLengthAdvisor(int order) {
        this.order = order;
    }
}
```

#### Step 2: 기본 메서드 구현
```java
@Override

    // TODO: 클래스의 단순 이름 반환
    return this.getClass().getSimpleName();
}

@Override

    // TODO: order 필드 반환
    return this.order;
```

#### Step 3: adviseCall 메서드 구현
```java
@Override
public ChatClientResponse adviseCall(ChatClientRequest request, CallAdvisorChain chain) {
    // TODO: 1. 요청 메시지 강화 (augmentPrompt 호출)
    ChatClientRequest mutatedRequest = augmentPrompt(request);
    
    // TODO: 2. 다음 Advisor 호출
    ChatClientResponse response = chain.nextCall(mutatedRequest);
    
    // TODO: 3. 응답 반환
    return response;
```

#### Step 4: 프롬프트 강화 메서드 구현
```java
private ChatClientRequest augmentPrompt(ChatClientRequest request) {
    // TODO: 1. 기본 안내 텍스트 생성
    String userText = this.maxCharLength + "자 이내로 답변해주세요.";
    
    // TODO: 2. Context에서 커스텀 값 가져오기
    Integer maxCharLength = (Integer) request.context().get(MAX_CHAR_LENGH);
    if (maxCharLength != null) {
        userText = maxCharLength + "자 이내로 답변해주세요.";
    }
    String finalUserText = userText;
    
    // TODO: 3. 기존 Prompt에 사용자 메시지 추가
    Prompt originalPrompt = request.prompt();
    Prompt augmentedPrompt = originalPrompt.augmentUserMessage(
        userMessage -> UserMessage.builder()
            .text(userMessage.getText() + " " + finalUserText)
            .build()
    );
    
    // TODO: 4. 수정된 ChatClientRequest 생성 및 반환
    ChatClientRequest mutatedRequest = request.mutate()
        .prompt(augmentedPrompt)
        .build();
    
    return mutatedRequest;
}
```

```java
@Service
@Slf4j
public class MaxCharLengthService {
    private ChatClient chatClient;

    public MaxCharLengthService(ChatClient.Builder chatClientBuilder) {
        this.chatClient = chatClientBuilder
            .defaultAdvisors(new MaxCharLengthAdvisor(Ordered.HIGHEST_PRECEDENCE))
            .build();
    }

    public String advisorContext(String question) {
        String response = chatClient.prompt()
            .advisors(advisorSpec ->

            .user(question)
            .call()
            .content();
        return response;
    }
}
```

## 🎓 실습 3: Advisor Bean 등록 및 Service 구현

### 목표
구현한 Advisor들을 Spring Bean으로 등록하고, Service에서 동적으로 활용하는 방법을 학습합니다.

### 실습 단계

#### Step 1: ChatClientConfig 구현
Spring Configuration에서 Advisor를 Bean으로 등록합니다.

```java
package com.example.springai.config;

import com.example.springai.advisor.SimpleLoggingAdvisor;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class ChatClientConfig {

    // TODO: SimpleLoggingAdvisor를 Bean으로 등록
    @Bean
    public SimpleLoggingAdvisor simpleLoggingAdvisor() {
        return new SimpleLoggingAdvisor();
    }
}
```

#### Step 2: AdvisorService 구현
SimpleLoggingAdvisor를 동적으로 적용하는 Service를 구현합니다.

```java
package com.example.springai.service;

import com.example.springai.advisor.SimpleLoggingAdvisor;
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.stereotype.Service;

/**
 * Advisor 기능을 테스트하는 서비스 클래스.
 * SimpleLoggingAdvisor를 동적으로 적용하여 LLM 호출 시 로깅합니다.
 */
@Service
public class AdvisorService {

    private final ChatClient.Builder chatClientBuilder;
    private final SimpleLoggingAdvisor simpleLoggingAdvisor;

    // TODO: 생성자를 통한 의존성 주입
    public AdvisorService(ChatClient.Builder chatClientBuilder, 
                         SimpleLoggingAdvisor simpleLoggingAdvisor) {
        this.chatClientBuilder = chatClientBuilder;
        this.simpleLoggingAdvisor = simpleLoggingAdvisor;
    }

    /**
     * TODO: SimpleLoggingAdvisor를 사용하여 간단한 텍스트 형식으로 로깅
     */
    public String callWithSimpleLogging(String userMessage) {
        ChatClient chatClient = chatClientBuilder
                .defaultAdvisors(simpleLoggingAdvisor)
                .build();
        
        return chatClient.prompt()
                .user(userMessage)
                .call()
                .content();
    }
}
```
