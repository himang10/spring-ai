# Spring AI RAG 실습 가이드

## 1. 실습 목표

이 가이드는 Spring AI 1.1.0에서 제공하는 RAG 도구 세트를 사용해 **Pre-Retrieval → Retrieval → Post-Retrieval** 전 과정을 단계별로 실습할 수 있도록 구성했습니다. 각 단계는 실제 코드(`QuestionAnswerService`, `RetrievalRAGService`, `QueryTransformerService`, `MultiQueryExpanderService`, `PostRetrievalRAGService`)와 HTTP 엔드포인트로 연결되어 있습니다. 

## 2. 사전 준비

```bash
export OPEN_AI_KEY=sk-...

mvn clean install -DskipTests
mvn spring-boot:run
```

- 접속 URL: `http://localhost:8080`
- 기본 문서는 `src/main/resources/documents/대한민국헌법(19880225).*` 에 위치하며, 애플리케이션이 `rag_vector_store` 테이블로 자동 적재합니다.

## 3. 전체 아키텍처 훑기

```
사용자 → Controller → ChatClient
         ↳ (선택) Query Transformer / Multi Query Expander
         ↳ VectorStoreDocumentRetriever (PGVector)
         ↳ (선택) DocumentPostProcessor
         ↳ ContextualQueryAugmenter → OpenAI Chat → 응답
```

## 4. 실습 시나리오

### 4.1 빠른 검증: `QuestionAnswerService`

- 파일: `src/main/java/com/example/springai/service/QuestionAnswerService.java`
- 엔드포인트: `POST /chat/qa`
- 사용 컴포넌트: `QuestionAnswerAdvisor`, `SearchRequest(topK=1, similarity=0.3)`

```bash
curl -X POST http://localhost:8080/chat/qa \
  -d "question=대한민국 헌법 제1조 1항의 내용은 무엇인가요?"
```

**실습 미션**
1. `topK` 값을 3으로 늘리고 답변 변화를 확인합니다.
2. `SimpleLoggerAdvisor`를 추가하여 Advisor 내부 프롬프트를 콘솔에서 확인합니다.

### 4.2 Retrieval 단계 이해: `RetrievalRAGService`

- 파일: `service/RetrievalRAGService.java`
- 엔드포인트: `POST /chat/rag/retrieval`
- 핵심 코드: `VectorStoreDocumentRetriever.builder().topK(3).similarityThreshold(0.3)`

```bash
curl -X POST http://localhost:8080/chat/rag/retrieval \
  -d "question=대한민국 헌법 제1조 2항의 내용은 무엇인가요?"
```

**실습 미션**
1. `similarityThreshold`를 0.5로 높여 검색 결과가 줄어드는지 확인합니다.
2. `topK`를 1로 낮춰 요약 품질 변화를 비교합니다.

### 4.3 Pre-Retrieval: `QueryTransformerService`

- 엔드포인트: `POST /chat/rag/pre-retrieval`
- 구성: `RewriteQueryTransformer` → `VectorStoreDocumentRetriever`

```bash
curl -X POST http://localhost:8080/chat/rag/pre-retrieval \
  -d "question=국회의원이 하라는 일은 하지 않고, 자기 개인 이익만 챙기고 있고 이게 국회의원이 할일이냐?"
```

**실습 미션**
1. `RewriteQueryTransformer`에 붙어 있는 `SimpleLoggerAdvisor` 로그를 확인해 재작성된 프롬프트를 살펴봅니다.
2. 특정 질문(예: "헌법 개정 조건?")으로 Pre-Retrieval 변환을 거친 답변이 Retrieval Only와 어떻게 다른지 비교합니다.

### 4.4 Multi Query Expansion: `MultiQueryExpanderService`

- 엔드포인트: `POST /chat/rag/multi-query-expander`
- 구성: `MultiQueryExpander`(기본 3개 변형) + `VectorStoreDocumentRetriever(topK=5)`

```bash
curl -X POST http://localhost:8080/chat/rag/multi-query-expander \
  -d "question=대통령의 임기는 어떻게 되나요?"
```

**실습 미션**
1. `MultiQueryExpander.builder().n(5)`로 수정하여 확장 쿼리 수를 늘리고 로그 변화를 확인합니다.
2. 로그에 출력된 각 변형 쿼리를 직접 Vector Store 검색에 사용해 보고, 실제로 추가 검색 결과가 늘어나는지 확인합니다.

### 4.5 Post-Retrieval 파이프라인: `PostRetrievalRAGService`

- 엔드포인트: `POST /chat/rag/post-retrieval`
- 구성 흐름:
  1. **Pre-Retrieval**: `MultiQueryExpander`
  2. **Retrieval**: `VectorStoreDocumentRetriever`
  3. **DocumentPostProcessor**: `score >= 0.5` 문서만 유지
  4. **Generation**: `ContextualQueryAugmenter` (`allowEmptyContext=true`)

```bash
curl -X POST http://localhost:8080/chat/rag/post-retrieval \
  -d "question=국회의 임시회의는 어떤 조건에 개최될수 있나요?"
```

**실습 미션**
1. DocumentPostProcessor에서 필터 기준을 0.4로 낮춘 뒤 답변 품질 변화를 확인합니다.
2. `ContextualQueryAugmenter`의 `allowEmptyContext`를 `false`로 두고 검색 결과가 없을 때 어떤 예외가 발생하는지 관찰합니다.
3. `DocumentPostProcessor` 람다에 재순위화 로직(예: 상위 2개만 유지)을 추가해보고 답변이 더 간결해지는지 확인합니다.

## 5. 프런트엔드 활용

- `src/main/resources/templates/home.html`의 드롭다운에서 전략을 선택하면 `chat.js`가 해당 엔드포인트로 `fetch` 요청을 전송합니다.
- JSON 응답은 `pre` 태그에 pretty-print되어 표시되므로, Multi Query와 Post-Retrieval 파이프라인의 결과를 UI에서 바로 확인할 수 있습니다.

## 6. 문제 해결 팁

| 현상 | 점검 항목 |
|------|-----------|
| `401 Unauthorized` | `OPEN_AI_KEY` 환경 변수가 설정되었는지 확인 |
| `relation "rag_vector_store" does not exist` | PostgreSQL 컨테이너 실행 상태와 `initialize-schema: true` 여부 확인 |
| 검색 결과가 항상 비어 있음 | 문서가 로드되었는지, `similarityThreshold`가 너무 높지 않은지 확인 |
| Multi Query 로그가 보이지 않음 | `MultiQueryExpanderService` 내부의 `SimpleLoggerAdvisor`가 주석 처리되지 않았는지 확인 |
